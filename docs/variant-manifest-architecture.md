# contenir-asset — Variant Manifest Architecture

**Status:** Phase 0 design (hardened) · branch `feature/variant-manifest`
**Date:** 2026-06-04
**Target release:** `contenir/contenir-asset` **2.0.0** (current: 1.0.4)
**Supersedes:** the earlier draft of this document (pre-council). This version reflects the thin-layer-on-storage decision, synchronous-with-guards generation, copy-forward migration, the *choice-first* framing, and the fixes from two review rounds.

---

## 1. Problem

Across the Contenir sites, image variants (resized AVIF/WebP derivatives for responsive `<picture>`/`srcset`) are generated by convention and **trusted** to exist, with no record of what exists and no per-variant dimensions. Four root problems follow:

1. **Broken `<picture>` sources** — a `<source>` the browser commits to by `type` does *not* fall back to `<img>` on a 404, so a missing derivative is a broken image, not graceful degradation.
2. **Layout shift (CLS)** — dimensions are never stored, so no aspect-ratio box can be reserved.
3. **Config drift** — phantom variants (a template requests `feature`, nothing generates it).
4. **Orphaned files** — obsoleted/superseded derivatives accumulate with no cleanup.

## 2. Design principle: choice

We are **not** picking one delivery model. The `StorageInterface` supports two adapter families and each site/profile chooses:

- **Pre-materialising (R2/S3, local FS)** — derivatives are real objects; this package's manifest records them. Full control, no per-request transform cost, backend-portable, no vendor lock-in. **This is what 2.0.0 builds.**
- **On-the-fly (Cloudflare edge resize, image proxy)** — derivatives are resolved at request time from one original. Already implemented (`storage/.../CloudflareImages.php`); existence is endpoint-guaranteed, so the manifest is **near-vestigial** here.

The manifest layer must therefore **degrade cleanly** for on-the-fly backends rather than assume pre-materialisation everywhere (see §6.4). Choosing the R2 path buys independence from metered edge resizing; the cost is the reconciliation machinery this document specifies.

## 3. Thin layer on storage

`contenir/storage` owns the **registry** and the **byte/resize mechanics**. `contenir-asset` adds the **record of truth** and the **read/heal/migrate domain** on top. Dependency: `asset → storage`, never the reverse.

```
contenir-asset (2.0.0)
  ├─ AssetMetadata (VO)     the manifest: sole writer of the JSON column
  ├─ VariantUrlResolver     pure: manifest rendition → URL (no network, no throw)
  ├─ AssetVariants/Picture  view helpers (fail-open)
  ├─ Reconciler             fused heal + GC (laminas-cli)
  └─ Migration              dual-mode resolver, legacy strategy, backfill, gates
        ↓ consumes
contenir/storage  VariantRegistry (owner) · generation · StorageInterface
```

**Storage enhancements required** (detailed in `storage-change-spec.md`): per-rendition `ImageMeta` return from generation; a pure, throw-free, network-free URL builder; a per-asset namespace layout + `list(namespace)`/`address`/`delete(Location)`/`deleteMany`; a write-response checksum; ingest guards in the resizer; clock/lock/verify seams. These are a precedent storage release that 2.0.0 pins to.

## 4. Manifest data model

Original metadata are **real, queryable columns**: `path`, `width`, `height`, `mime`, `filesize`. Derivatives live in a JSON column, wrapped exclusively by the `AssetMetadata` VO.

```jsonc
{
  "v": 2,                                  // schema version — branch point for future shapes
  "def": "a1b2c3",                         // registry/definition hash this manifest was built against
  "renditions": {
    "card": [                              // FLAT list per variant — NOT keyed by format
      { "format": "avif", "mime": "image/avif", "media": null,
        "width": 600, "height": 600, "filesize": 18342,
        "key": "asset/812/card.9f3a.avif", "hash": "9f3a…", "generated_at": 1717459200 },
      { "format": "webp", "mime": "image/webp", "media": null,
        "width": 600, "height": 600, "filesize": 22104,
        "key": "asset/812/card.7b1e.webp", "hash": "7b1e…", "generated_at": 1717459200 }
    ],
    "hero": [
      { "skipped": "source_smaller_than_target", "format": "avif" }   // deliberate skip, not a gap
    ]
  },
  "meta": {}                               // reserved extensibility bag (blurhash, focal point, dominant colour)
}
```

Design notes (each a council fix):
- **`"v"` schema-version** — a v3 shape change branches in `fromJson()` without a fleet rewrite; absent/unknown version fails open.
- **Flat `renditions[]` per variant, not format-keyed** — format-as-key can't represent same-MIME variants (HDR vs SDR), animation, or art-directed `media`. Each rendition carries its own `format`/`mime`/optional `media`.
- **`def` hash** — lets reconcile detect *redefinition* (a variant's dims/fit changed) as a heal delta, not just absence.
- **`hash` per rendition** — content hash; doubles as the versioned-key token (immutable caching) and the write-verify validator.
- **`skipped` sentinels** — `enlarge=no` on a small original is a *deliberate* non-rendition; without this the completeness gate (§6.3) blocks the manifest-only flip forever.
- **`meta` bag** — round-tripped untouched so a downstream package can add LQIP/blurhash/focal-point without forking the VO.
- **Per-rendition real dims** — variants have different crops/ratios (`card` 600×600 cover vs `hero` contain); dims are the resizer's *actual output*, never derived.

`AssetMetadata::fromJson(?string): self` **never throws** (garbage/null → empty set = fail-open). Rendition VOs validate in their constructor (positive-int dims, non-empty key, known format) so an invalid manifest is unrepresentable.

## 5. Write path (synchronous, with guards)

On upload/edit, generation runs **in-request with guards** (no queue dependency):

1. **Ingest guard** (in storage's resizer): raster MIME allowlist; `-auto-orient` (fixes EXIF dim-swap → CLS); `-strip`; sRGB; `policy.xml` memory/area/time limits; **Imagick ext or `proc_open` argv array — never a shell string**; pixel/area + wall-clock caps. Non-raster (SVG/PDF/video) → pass-through: store original, empty manifest.
2. **Generate outside the lock**, with a hard wall-time budget *below* the web timeout. On budget exhaustion, commit **nothing** (asset stays on original/legacy) — never a partial manifest; surface "processing, refresh shortly", not a 504.
3. **Verify by write-response checksum** (ETag/strong validator compared to the locally computed content hash) — *not* a separate HEAD (HEAD-200 doesn't prove byte integrity and costs a round-trip). Store the hash in the rendition.
4. **Commit last, under the lock only**: acquire the per-asset advisory lock with a short timeout, **manifest-version CAS**, write the complete `AssetMetadata` + original columns in one DB transaction, release. Lock contention → fail the *write* cleanly ("busy"), don't block a worker for the whole generation.

The read path's 404-tolerance (always emit the original `<img>`) remains the real safety net; the manifest is authoritative-but-advisory.

## 6. Read path

### 6.1 Helpers
- **`AssetVariants($asset, $key) → RenditionSet`** — typed, markup-free; the reuse seam (helpers, JSON, controllers, CLI). Built from `AssetMetadata::renditionsFor()` + storage's pure `url(Location)`; zero network.
- **`AssetPicture($asset, [...])`** — emits `<picture>`/`<img>`. Separate `picture`/`img` attribute bags; helper-owned reserved attrs (`src`/`srcset`/`width`/`height`/lazy markers) win on collision; first-class `sizes`; eager/lazy toggle. Every attr `escapeHtmlAttr`'d. Box/`aspect-ratio` from the **displayed rendition's** real dims.

### 6.2 Fail-open cascade (the safety floor)
Emit only manifest-vouched `<source>`s → always emit `<img src=original>` → partial/absent/corrupt/unparseable manifest → original only → empty `path` → render nothing. Never throw, probe, or touch the network at read time.

### 6.3 Completeness — defined against the registry
An asset is **complete** iff, for every registry variant × format applicable to its source MIME, a rendition (or a `skipped` sentinel) exists. It is a **pure function of (manifest, registry def-hash)** — *not* per-template (the per-template framing was self-contradictory; reconcile generates the **union** of all registry variants, never a per-template subset). Completeness gates the `manifest-only` flip (§8).

### 6.4 On-the-fly degeneracy
For an edge backend, `AssetVariants` collapses to a single negotiated rendition per variant (one `<img srcset>`, no per-format `<source>`) and the manifest is bypassed. This keeps "move a site to derivation later" a config flip, not a read-path rewrite.

## 7. Reconcile (fused heal + GC, one `laminas-cli` command)

```
reconcile(asset):                          # idempotent, resumable, per-asset locked
  expected = registry × original           # union of all registry variants for the source MIME
  present  = storage.list(namespace)       # authoritative; NOT the keyExists cache
  generate missing → write+verify          # heal (outside lock)
  rewrite  manifest (CAS, in lock)         # repoint at new keys
  prune    orphans = present − (expected ∪ original), AND aged   # GC (last)
```

Gates (every one a fix):
- **Dry-run default**; `--prune` to delete; `--no-prune` for heal-only.
- **Prune only in `manifest-only` mode** (no-op otherwise, independent of `--prune`).
- **Registry-version pin distinguishing additive vs mutating** — adding a variant (additive) proceeds and just generates the new one; a *mutating* change (dims/fit on an existing variant) refuses and requires re-backfill. Pin is a content-hash of the variant set.
- **Per-asset AND fleet-level orphan kill switch** — fleet state in a shared row (`tripped_at`, reason, prune budget); each delete decrements a shared budget and re-checks the flag; explicit operator reset.
- **Retention: hard floor + reference check** — never delete by age alone; prune only if the key is absent from the current manifest **and** older than a hard floor (default 30d, treating HTML/CDN TTL as unbounded unless an operator declares a max) **and** not in the live reference set (which **unions the original `path`**).
- **Poison-pill quarantine** after N failed heals; quarantined and missing-original assets are **excluded from the completeness denominator** (with an alarm count) so one bad asset can't pin the fleet, and stop re-alarming.
- **Per-delete audit** to an append-only sink: key, reason, age, operator identity, run-id, the manifest version checked against.

## 8. Migration — copy-forward into new buckets

- **New namespaced bucket per site; old bucket read-only** as the migration source **and** instant rollback. We never mutate live assets.
- **Per-site mode flag** `legacy | dual | manifest-only`; **dual-mode resolver** (manifest-first, pluggable `LegacyVariantStrategyInterface` fallback). Relative-key storage → swapping buckets is a per-profile `publicUrl` change.
- **Additive nullable schema** (`variants` JSON + original columns). Empty column ⇒ no behaviour change.
- **Backfill re-ingests from the original** (generate fresh, capture real dims — no probe-and-trust). Resumable cursor; atomic per-asset manifest write.
- **Derived-only legacy assets** (no original): promote the best available derived image to a *synthetic* original (`original_synthetic=true`), generate downscale-only variants, exclude from the completeness denominator.
- **Writes during migration go to the new bucket only**; old bucket is strictly historical. Record migration-start; rollback explicitly forfeits assets created after it (warn).
- **`manifest-only` activation is a blocking precheck** — refuse the mode change unless zero assets are incomplete (per §6.3). Not operator discipline.
- **GC stays disabled until `manifest-only`** and fully backfilled.

## 9. Observability, ops, security

- **Observability is a contract, not prose:** inject PSR-3 + a metrics sink. Emit `variant.generated`, `variant.generate_failed` (with captured stderr — no more `2>/dev/null`), `reconcile.healed{count}`, `reconcile.pruned{key,reason,age}`, `manifest.parse_failed`, `manifest.key_404` (edge rate). Define the alert threshold (heal-rate > N/min over baseline).
- **`asset:status [--site]`** — read-only: per-site mode, % assets with complete manifests, phantom/unregistered-variant count, orphan estimate. The single command an operator runs before flipping `manifest-only`.
- **Security:** SVG is sanitised on ingest (`enshrined/svg-sanitize`) **or** served `Content-Disposition: attachment` + CSP from a cookieless host — not left in an unexamined pass-through. `reconcile --prune` runs under a restricted role, separate from the web user, with an explicit confirmation token for fleet prune; audit log is append-only/tamper-evident. Ingest accepts uploaded bytes only (no URL-fetch → no SSRF) until an allowlisted, DNS-rebind-safe fetch path is ever added.

## 10. Packaging, semver, dependencies

- PHP floor **8.2** (matches storage's `readonly`/enums); phpunit **^10/^11**.
- `require` `contenir/storage:^X.Y` **pinned to the interface-introducing minor**, `conflict` older; CI matrix runs `--prefer-lowest` and highest. Every storage interface bump pairs with a documented asset range (`UPGRADING.md`).
- `laminas-mvc` → optional; require `laminas-servicemanager` + `laminas-cli`; suggest mvc/mezzio. `ConfigProvider` + `Module` for both; factories **inject the app's resolved `StorageManager`** (asset doesn't know the profile/creds).
- Consider whether `contenir-db-model` is core or sits behind a repository interface so the VO isn't married to one ORM.
- Static analysis (psalm/phpstan max), `phpcs` (Laminas standard), and **Infection mutation testing scoped to the prune/age/breaker predicates** (a surviving mutant there = a silent data-loss path). S3 integration tests against MinIO in CI; real-R2 smoke gated to nightly.
- Tag `1.x` maintenance line before 2.0 work; ship `2.0.0`.

## 11. Forward-compatibility policy

- **Manifest schema** evolves via `"v"` + a `fromJsonVN()` upgrader; never an in-place fleet rewrite.
- **Adding a format** is additive renditions. **Retiring a format** is two-phase: stop emitting in the read path → wait ≥ retention/cache TTL → drop from registry → reconcile prunes. (Mirror of the addition path; absent it, the first retirement risks vouched-but-deleted sources.)
- **Versioned-key token standardises on content-hash** (`generated_at` is metadata only) so `Cache-Control: immutable` holds for the key's lifetime.
- **Multi-tenant/per-brand** variant sets: keep a `RegistryProvider` indirection (resolvable by key/tenant) even with one registry today — cheap seam, expensive retrofit.

## 12. Invariants

1. `VariantRegistry` (in storage) is the single source of truth; asset validates against it — no second registry.
2. `AssetMetadata` VO is the sole writer of the manifest column; readers never touch raw JSON.
3. The manifest records the **verified-produced**, never the config-intended; verification is by write-response checksum.
4. Read path fails open to the original; never throws, probes, or touches the network.
5. Completeness is a pure function of (manifest, registry def-hash), asset-level, registry-wide.
6. Namespace is exclusive per asset — the precondition for safe GC; reconcile asserts the original appears in its own namespace listing before pruning.
7. Generation happens outside the lock; only the CAS commit is locked.
8. Prune never deletes by age alone, never outside `manifest-only`, never past the fleet budget.

## 13. Open decisions

- DB engine per site (Postgres `jsonb` + CHECK vs MySQL/MariaDB JSON) — how much manifest integrity is DB-enforced vs VO-enforced.
- Exact retention floor + whether any site can declare a bounded max-HTML-TTL.
- Whether `contenir-db-model` stays a hard dep or hides behind a repository interface.
- Nearest-available rendition fallback (deferred; original-as-floor is the simple always-correct rule).

## 14. Non-goals (for 2.0.0)

- Art-directed `<picture>` (different crops per breakpoint) — the flat-rendition `media` field *reserves* for it; the helper layer ships later.
- Replacing `contenir/storage` — it stays the byte/resize engine and registry owner.
- A runtime CMS↔FE variant-negotiation protocol — shared config (the registry) is the agreement.
- A URL-fetch ingest path — uploaded bytes only.

## Appendix A — reality-check: what storage does NOT have yet

The plan depends on storage capabilities that are **unbuilt** today (grounded in the current `contenir/storage` source) — tracked in `storage-change-spec.md`:

- No `address`/`list(namespace)`/`delete(Location)`/`url(Location)`/Location surface; no per-asset namespace (variants are siblings in shared dirs → today's GC safety proof is false).
- `ImageResizer` shells out via `exec(… 2>/dev/null)` with no timeout/policy/caps/auto-orient → the ingest guards are unbuilt.
- No clock/lock/verify seams; timestamps from bare `new DateTimeImmutable()`.
- No logger/metrics anywhere; failures are swallowed.
- `InMemoryStorage::regenerateMissingVariants()` is a no-op → can't exercise the heal path; the `knownKeys` positive cache can make reconcile's diff lie.

## Appendix B — ships model-agnostic, regardless of the above

These are good under *any* model, independently shippable, reversible, low-risk — land them early to de-risk:
1. Write the original's `width/height/mime/filesize` columns at upload — the real CLS fix.
2. One registry as the single source of truth — kills phantom variants.
3. The fail-open `<picture>` helper — emit only vouched sources, always emit the original.
